# Contributing to Car Company Website
Thank you for considering contributing to the Car Company Website project. This guide outlines the steps and expectations for contributing to our open-source project.

## Getting Started
To contribute to the Car Company Website project, you'll need to fork and clone the repository. Here's how:

1. **Fork the repository**: Navigate to the [Car Company Website repository](https://github.com/your-username/Car-Company-Website) and click the "Fork" button in the top-right corner.
2. **Clone the repository**: Run the following command in your terminal, replacing `your-username` with your actual GitHub username:
   ```bash
git clone https://github.com/your-username/Car-Company-Website.git
```
3. **Navigate to the repository**: Change into the cloned repository:
   ```bash
cd Car-Company-Website
```

## Branching
When making changes, create a new branch to isolate your work. We follow a standard naming convention for branches:

* **Feature branches**: Use the prefix `feature/` followed by a brief description of the feature (e.g., `feature/project-specification-generation`).
* **Bug fix branches**: Use the prefix `fix/` followed by a brief description of the fix (e.g., `fix/repository-scaffolding-issue`).
* **Documentation branches**: Use the prefix `docs/` followed by a brief description of the documentation change (e.g., `docs/update-readme`).

Create a new branch using the following command:
```bash
git checkout -b feature/your-branch-name
```
Replace `your-branch-name` with the actual name of your branch.

## Pull Requests
When you're ready to submit your changes, create a pull request against the `main` branch. Here's what we expect from a pull request:

* **Clear title**: Use a concise and descriptive title that summarizes the changes.
* **Detailed description**: Provide a detailed description of the changes, including any relevant context or explanations.
* **Related issues**: If your pull request fixes an issue, include the issue number in the description (e.g., "Fixes #123").
* **Code quality**: Ensure your code adheres to our code style expectations (see below).

## Code Style Expectations
We follow standard professional guidelines for code style:

* **Indentation**: Use 4 spaces for indentation.
* **Line length**: Keep lines under 80 characters.
* **Comments**: Use clear and concise comments to explain complex code.
* **Commit messages**: Follow the standard GitHub guidelines for commit messages.

## Issue Reporting
If you encounter an issue or bug, please report it using our issue template:

1. **Create a new issue**: Navigate to the [Issues](https://github.com/your-username/Car-Company-Website/issues) page and click the "New issue" button.
2. **Use the template**: Select the "Bug report" template and fill in the required information, including:
	* A clear and concise title.
	* A detailed description of the issue.
	* Steps to reproduce the issue (if applicable).
	* Any relevant error messages or logs.

By following these guidelines, you'll help us maintain a high-quality and consistent codebase. Thank you for your contributions to the Car Company Website project!
