# CONTRIBUTING.md
## Introduction
Thank you for considering contributing to the "Website Showing All Products Of Uniliver In Pakistan" project. This project aims to create a comprehensive website showcasing all Uniliver products available in Pakistan. We appreciate your interest in helping us achieve this goal.

## Getting Started
To contribute to this project, follow these steps:

### Forking and Cloning the Repository
1. **Fork** the repository by clicking the "Fork" button in the top-right corner of the repository page.
2. **Clone** the forked repository to your local machine using the command:
   ```bash
git clone https://github.com/your-username/Website-Showing-All-Products-Of-Uniliver-In-Pakistan.git
```
3. **Navigate** to the cloned repository:
   ```bash
cd Website-Showing-All-Products-Of-Uniliver-In-Pakistan
```

### Branching
Create a new branch for your feature or bug fix using the following naming convention:
```
feature/your-feature-name
fix/your-fix-name
```
For example:
```bash
git checkout -b feature/add-product-filter
```
or
```bash
git checkout -b fix/resolve-pagination-issue
```

## Pull Request Expectations
When submitting a pull request, please ensure that:

* Your code is formatted according to our code style guidelines (see below).
* Your changes are thoroughly tested and do not introduce any regressions.
* Your pull request includes a clear and concise description of the changes made.
* Your pull request is targeted at the `main` branch.

## Code Style Expectations
We follow standard professional guidelines for code style. Please ensure that your code adheres to the following:

* Use a consistent naming convention (e.g., camelCase or underscore notation).
* Keep functions and methods concise and focused on a single task.
* Use comments to explain complex logic or algorithms.
* Use whitespace and blank lines to improve code readability.

## Issue Reporting Guidance
When reporting an issue, please provide the following information:

* A clear and concise description of the issue.
* Steps to reproduce the issue (if applicable).
* Any relevant error messages or logs.
* Your environment and setup (e.g., browser, operating system, etc.).

You can report issues using the GitHub issue tracker. When creating a new issue, please use the following format:
```
### Issue Description
* Brief summary of the issue
* Steps to reproduce (if applicable)
* Error messages or logs (if applicable)
* Environment and setup
```
Example:
```
### Issue Description
* The product filter is not working as expected
* Steps to reproduce: 
  1. Navigate to the products page
  2. Select a filter option
  3. The filter does not apply
* Error messages or logs: None
* Environment and setup: Google Chrome, Windows 10
```

## Commit Message Guidelines
When committing changes, please follow the standard GitHub guidelines for commit messages:

* Use the imperative mood (e.g., "Add" instead of "Added").
* Keep the first line concise and focused on the main change.
* Use a blank line to separate the brief summary from the body.
* Use bullet points to break up large blocks of text.

Example:
```
Add product filter feature

* Implement filter logic
* Update UI to reflect filter options
* Test filter functionality
```

By following these guidelines, you can help us maintain a high-quality and consistent codebase. Thank you for your contributions to the "Website Showing All Products Of Uniliver In Pakistan" project!
